Usage

Run the plugin on demand

Declare in your maven project:

<project>
  ...
  <build>

    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>com.github.amanganiello90</groupId>
        <artifactId>manage-pom-maven-plugin</artifactId>
        <version>1.0<version>
      </plugin>
      ...
    </plugins>
  </build>
  ...
</project>

Or modify adding in your maven settings.xml :

...
<pluginGroups>        
        <pluginGroup>com.github.amanganiello90</pluginGroup>
</pluginGroups>
...

{warning} With the pluginGroups configuration you use the last release version available on maven central. {warning}

Now you can run the plugin with:

mvn manage-pom:<goal>

Where <goal> is a specified plugin goal, for example:

mvn manage-pom:check-snapshots

Run the plugin in a maven phase

If you want to automatically run a specified plugin goal (for example the check-snapshots) in a maven phase (i. e. package phase), declare in your maven project:

<project>
  ...
  <build>

    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>com.github.amanganiello90</groupId>
        <artifactId>manage-pom-maven-plugin</artifactId>
        <version>1.0<version>
        <executions>
			<execution>
			  <id>check-snapshot-package-phase</id>
			  <phase>package</phase>
			  <goals>
				<goal>check-snapshots</goal>
			  </goals>
			</execution>
	</executions>
       <configuration>
          ...
       </configuration>
      </plugin>
      ...
    </plugins>
  </build>
  ...
</project>

For more informations, see "Guide to Configuring Plug-ins".
